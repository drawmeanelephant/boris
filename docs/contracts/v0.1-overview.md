## Product statement

Boris v0.1 is a **Zig content compiler**: discover Markdown → bounded frontmatter
→ Trunk/Satellite graph validation → deterministic JSON IR under `.boris/` (or
`--out`). Optional product RAG (`--rag` / `--rag-dir`) and **opt-in HTML** site
mode (`--html` / `--html-dir` / `--target`; Apex + Whiteboard) share the same
graph rules. IR/RAG and pre-render coordination remain sequential; HTML page
render may use bounded `--jobs` workers. HTML is **not** the bare-`boris`
default CLI product surface for v0.1 acceptance.

Compiler id: `boris/0.1.1`. IR `schemaVersion`: `0.1.0`.

## Author-facing rules (summary)

| Topic | Rule |
|-------|------|
| Frontmatter | Closed grammar; not YAML — [frontmatter.md](frontmatter.md) |
| Keys | Exactly `id`, `title`, `parent`, `status`, `tags` |
| Parent key | **`parent` only** — never `parentEntry` |
| Trunk | No `parent` |
| Satellite | Exactly one `parent` naming a Trunk id |
| Paths / ids | `/` separators; no absolute, empty, `.`, or `..` segments — [identity-and-paths.md](identity-and-paths.md) |
| Extensions | Case-sensitive `.md` and `.mdx` only — [scanner.md](scanner.md) |
| Default output | Deterministic JSON IR under `.boris/` — [ir-schema.md](ir-schema.md) |
| Asides | Constrained `<Aside>` on shared compile path; RAG may export `:::kind` (not authoring) — [components.md](components.md), [rag-export.md](rag-export.md) |
| HTML | Opt-in CLI (`--html` / `--html-dir` / `--target`) — [html-output.md](html-output.md) |
| RAG | Optional product export — [rag-export.md](rag-export.md) |
