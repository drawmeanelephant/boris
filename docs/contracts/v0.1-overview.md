# v0.1 overview — content compiler (planning)

**Status:** high-level orientation. Normative detail lives in the five contracts
listed in [README.md](README.md).

## Product statement

Boris v0.1 compiles a Markdown content tree into a **validated, versioned,
deterministic intermediate representation (IR)** under `.boris/`. Downstream
tools (optional RAG export, experimental HTML) are consumers — not required for
IR acceptance.

## Author-facing rules (summary)

| Topic | Rule |
|-------|------|
| Frontmatter | Closed grammar; not YAML — [frontmatter.md](frontmatter.md) |
| Keys | Exactly `id`, `title`, `parent`, `status`, `tags` |
| Parent key | **`parent` only** — never `parentEntry` |
| Trunk | No `parent` |
| Satellite | Exactly one `parent` naming a Trunk id |
| Paths / ids | `/` separators; no absolute, empty, `.`, or `..` segments |
| Extensions | Case-sensitive `.md` and `.mdx` only |
| Default output | Deterministic JSON IR under `.boris/` |
| HTML | Not default product surface |
| RAG | Optional future export — [rag-export.md](rag-export.md) |
